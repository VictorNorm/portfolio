Semester project report.

This was the second project of the year, and this was the second website i've ever built in my life.

As usual the aspect of front end development i struggle most with is the design part, Id like to think that this website is more visually appealing than my first project. I'm happy with how the coloration, fonts and the hierarchy turned out. There are big improvements to be made in general around the site regarding functionality.

Here are some of the updates i've made during the last week.

  • -Fixed bug bug with images stretching in chrome.
  • -Fixed disappearance of image on index page
  • -Fixed bug with font imports.
  • -Removed placeholder text in inputs on the tickets page.
  • -Added text on smaller adds on experiences page
  • -Changed the structure for the smaller adds removing major stretching issues
  • -Different smaller aesthetic tweaks.

Fix image bug: The image on the front page stretched to 3 times the intended height in chrome, making the front page look awful

Fix disappearing image: The image on the front page vanished completely at lower screen sizes

Fix font bug: The syntax of the font importation was wrong, fixed this issue.

Removed placeholder text: I removed the placeholder text on the tickets page as this created unneccessary clutter on the page. There was no need for it.

Changed add structure: The adds didn't respond the way I originally intended them to, fixed this by making small tweaks, the goal here was to create a more responsive page.

Conclusion: The experience of looking back at this project is similar to looking back at the first project. There are lots of things that could have been done in more efficient way. For this project the CSS is the part that could have improved the most (this is quite obvious since it is the biggest part of the project).

I chose not to add any JS because I felt that time would be better spent improving the other 2 projects. Would I have been more efficient in development during this project I would have structured it differently. The naming conventions in the CSS i think are quite messy, the whole file could be a lot shorter. I think I also could have managed with fewer media queries.

If I started to make improvements with JS it would result in spending as much time as I did on the whole project, it might be that this is what you are after but I didn't percieve it as such.

Some of the things i know I would have added if I remade the whole project..

  • -A fully functional calendar.
  • -More fleshed out information about the events at the museum (preferably with videos).
  • -Animations for some of the CTA-buttons indicating that an action has been taken/recieved (example: when booking has been succesful)

There are still lots more to improve as there always is, but what I've learned from this project is: To make something more visually appealing, and most of all: CHECK that it works in different browsers, (can't believe I fell into this trap).
